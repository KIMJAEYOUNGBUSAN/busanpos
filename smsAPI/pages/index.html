smsAPI Agent

GitHub Pages에서 쓰는 API 안내 및 테스트 페이지

배포 구조

GitHub Pages는 정적 페이지 전용입니다. 이 화면은 API 사용과 테스트를 돕는 페이지이고, 실제 문자/이메일 발송 백엔드는 Render, Railway, Fly.io 같은 Node 호스팅에 배포해야 합니다.

브라우저 테스트용 API Key는 이 기기에만 저장됩니다. 운영 서비스에서는 서버에서 자체 API를 호출하는 방식을 권장합니다.

연결 설정

문자 테스트

이메일 테스트

상태 조회

응답

아직 요청하지 않았습니다.

API 예시

POST /v1/messages/sms
Authorization: Bearer 자체_API_KEY
Content-Type: application/json

{
  "to": "01012345678",
  "from": "0212345678",
  "text": "[서비스명] 인증번호는 123456입니다.",
  "clientRequestId": "signup-10001"
}

테스트 순서

  1. 백엔드를 Render, Railway, Fly.io 또는 VPS에 배포합니다.
  2. 백엔드 관리자 화면에서 솔라피와 Resend 키를 저장합니다.
  3. API 클라이언트를 만들고 자체 API Key를 복사합니다.
  4. 이 화면에 백엔드 URL과 자체 API Key를 입력한 뒤 연결 확인을 누릅니다.
  5. 문자 또는 이메일 테스트 발송을 실행합니다.
