# ChallengeOps MVP

블로그, 유튜브, 틱톡 챌린지를 운영하기 위한 웹 기반 MVP입니다.

## 현재 구현 범위

- 서버 기반 로그인/회원가입
- JSON 파일 데이터 저장소
- 관리자, 운영자, 참가자 역할 구분
- 챌린지 생성/수정
- 챌린지 탐색/상세/참가 신청
- 참가자 결제 상태 수동 관리
- 참가자 입금 확인 정보 제출
- 참가자 검색과 결제 상태 필터
- 미션 제출
- 인증 이미지 업로드와 제출물 썸네일 확인
- 제출물 상태 필터와 검색
- 제출물 승인/반려/보완 요청
- 선택 제출물 일괄 승인
- 진행률, 성공 여부, 랭킹 계산
- 대시보드 운영 리포트
- 랭킹 확정
- 지급 대상자 생성과 지급 상태 수동 관리
- 참가자, 제출물, 정산 CSV 다운로드
- 이의 신청
- 웹 알림
- 운영자 요금제/사용량 표시
- 관리자 플랜 수동 변경
- 관리자 전용 샘플 초기화
- 관리자 전용 JSON DB 백업/복원
- 브랜드 캠페인 문의
- 운영 로그
- 서버 액션 API 기반 상태 변경
- 주요 작업의 서버 권한 검증

## 실행

```bash
node server.js
```

브라우저에서 다음 주소로 접속합니다.

```text
http://localhost:3000
```

PowerShell 실행 정책 때문에 `npm` 명령이 막히는 환경에서는 `npm start` 대신 `node server.js`를 사용하면 됩니다.

## 테스트 계정

로그인 화면에서 테스트 버튼을 사용할 수 있습니다.

- 관리자: `admin@challenge.local` / `1234`
- 운영자: `brand@challenge.local` / `1234`
- 참가자: `user@challenge.local` / `1234`

## 데이터 저장

서버 실행 시 `data/challengeops-db.json` 파일이 자동 생성됩니다. 이 파일은 런타임 데이터이므로 Git 추적에서 제외됩니다.

미션 인증 이미지는 `uploads/submissions/` 아래에 저장됩니다. JPG, PNG, WEBP 파일을 10MB 이하로 업로드할 수 있으며, 저장된 파일은 제출물 목록과 검수 화면에서 썸네일로 확인할 수 있습니다. `uploads/` 역시 런타임 파일이므로 Git 추적에서 제외됩니다.

초기화는 관리자 계정에서만 노출되는 `샘플 초기화` 버튼으로 할 수 있습니다.

## 서버 API 구조

프론트엔드는 서버 모드에서 전체 상태를 직접 덮어쓰지 않고 `/api/action`으로 주요 작업을 요청합니다. 서버는 세션 사용자 역할과 챌린지 소유권을 확인한 뒤 JSON DB를 갱신합니다.

서버에서 검증하는 주요 작업:

- 챌린지 생성/수정
- 참가 신청
- 입금 확인 정보 제출
- 결제 상태 변경
- 미션 제출
- 제출물 검수
- 제출물 일괄 승인
- 랭킹 확정
- 지급 대상자 생성
- 지급 상태 변경
- CSV 다운로드 로그 기록
- 이의 신청과 답변
- 알림 읽음 처리
- 관리자 플랜 변경
- 관리자 DB 백업/복원

기존 전체 상태 저장 엔드포인트 `/api/state`는 보안상 폐쇄되었고 `410` 응답을 반환합니다.

## 주의

현재 버전은 실제 PG 결제, 자동 송금, 원천징수 계산, 외부 플랫폼 API 자동 검수를 제공하지 않습니다. PRD 기준에 따라 1차 제품은 수동 결제 상태 관리, 수동 지급 상태 관리, 검수/랭킹/정산 보조에 집중합니다.

## smsAPI Agent

`smsAPI/` 폴더에는 솔라피 문자 발송과 Resend 이메일 발송을 하나의 자체 API로 묶는 발송 에이전트가 포함되어 있습니다.

GitHub Pages 테스트 화면은 `docs/` 폴더에 준비되어 있습니다. 저장소에서 아래처럼 한 번만 설정하면 됩니다.

```text
Settings > Pages > Source: Deploy from a branch
Branch: master
Folder: /docs
```

Pages 주소가 생성되면 해당 화면에서 백엔드 URL과 자체 API Key를 입력해 문자/이메일 발송을 테스트할 수 있습니다. 실제 Node 백엔드는 GitHub Pages에서 실행되지 않으므로 Render, Railway, Fly.io, VPS 같은 별도 Node 호스팅에 배포해야 합니다.
